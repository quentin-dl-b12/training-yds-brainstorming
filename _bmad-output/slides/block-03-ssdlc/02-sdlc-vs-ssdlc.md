# SDLC → SSDLC: what changes

| Phase | Security activity added |
|---|---|
| Requirements | Security requirements, auth rules, data classification |
| Design | Threat modeling, architecture risk review |
| Implementation | Secure coding, secrets management, dependency hygiene |
| Testing | SAST, SCA, DAST, penetration testing |
| Deployment | Secure configuration, environment separation |
| Operations | Vulnerability monitoring, incident response, patching |

> "SSDLC is what happens when you stop treating security as a final gate and start treating it as a design property from day one."
