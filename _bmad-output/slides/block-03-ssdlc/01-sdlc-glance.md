# SDLC at a glance

```
Requirements → Design → Implementation → Testing → Deployment → Operations
```

| Phase | What it produces |
|---|---|
| Requirements | Specifications, user stories |
| Design | Architecture, data models |
| Implementation | Code, unit tests |
| Testing | Test reports, bug fixes |
| Deployment | Running system, release notes |
| Operations | Monitoring, patches, incident response |

**Modern reality:** Agile/DevOps compresses and repeats these phases — the lifecycle is a loop, not a waterfall.
